Kimi for Coding is a subscription-only provider with an OpenAI-compatible
API. GoModel handles model discovery automatically — the only available model
is `kimi-for-coding` — and sets ZooCode-identifying headers on every request so
upstream accepts the connection.

## Configure

```bash
KIMI_API_KEY=...
```

Or in `config.yaml`:

```yaml
providers:
  kimi:
    type: kimi
    base_url: "https://api.kimi.com/coding/v1"
    api_key: "${KIMI_API_KEY}"
```

<Note>
  `KIMI_BASE_URL` is optional and defaults to `https://api.kimi.com/coding/v1`.
</Note>

## Synthetic model list

Kimi does not expose a `/models` endpoint, so GoModel returns a synthetic list
containing exactly one model:

| Model ID | Description |
| -------- | ----------- |
| `kimi-for-coding` | Kimi for Coding subscription model |

No `KIMI_MODELS` configuration is required. The model is always available.

## Header spoofing

Kimi for Coding only accepts requests from permitted user agents. GoModel
automatically sets the following ZooCode-identifying headers on every upstream
request:

- `Content-Type: application/json`
- `Http-Referer: https://github.com/Zoo-Code-Org/Zoo-Code`
- `User-Agent: ZooCode/3.62.0`
- `X-Title: Zoo Code`

Standard browser/SDK metadata (`Accept-*`, `Connection`, `Sec-Fetch-Mode`,
`X-Stainless-*`) is intentionally omitted because an API server does not
validate them for access control.

No manual configuration is needed.
