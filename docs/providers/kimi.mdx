Kimi for Coding is a subscription-only provider with an OpenAI-compatible
API. GoModel handles model discovery automatically — the only available model
is `kimi-for-coding` — and sets ZooCode-identifying headers on every request so
upstream accepts the connection.

## Configure

```bash
KIMI_API_KEY=...
```

Or in `config.yaml`:

```yaml
providers:
  kimi:
    type: kimi
    base_url: "https://api.kimi.com/coding/v1"
    api_key: "${KIMI_API_KEY}"
```

<Note>
  `KIMI_BASE_URL` is optional and defaults to `https://api.kimi.com/coding/v1`.
</Note>

## Model discovery

Kimi exposes a `/models` endpoint, so GoModel automatically discovers its model
list from the upstream API. The provider currently returns one model:

| Model ID | Description |
| -------- | ----------- |
| `kimi-for-coding` | Kimi for Coding subscription model |

No `KIMI_MODELS` configuration is required. The model list is fetched from the
upstream API at runtime.

## Header spoofing

Kimi for Coding only accepts requests from permitted user agents. GoModel
automatically sets the following natural headers on every upstream request so
that traffic looks like a real browser/SDK:

- `Accept: application/json`
- `Accept-Encoding: gzip, deflate`
- `Accept-Language: *`
- `Content-Type: application/json`
- `Http-Referer: https://github.com/Zoo-Code-Org/Zoo-Code`
- `Sec-Fetch-Mode: cors`
- `User-Agent: ZooCode/3.62.0`
- `X-Stainless-Arch: x64`
- `X-Stainless-Lang: js`
- `X-Stainless-Os: Linux`
- `X-Stainless-Package-Version: 5.12.2`
- `X-Stainless-Retry-Count: 0`
- `X-Stainless-Runtime: node`
- `X-Stainless-Runtime-Version: v22.22.1`
- `X-Title: Zoo Code`

No manual configuration is needed.

## Passthrough

Kimi can be used through the passthrough API at `/p/kimi/...`.

<Note>
  If you use a custom `config.yaml`, add `kimi` to `enabled_passthrough_providers`:

  ```yaml
  server:
    enabled_passthrough_providers: ["openai", "anthropic", "openrouter", "zai", "vllm", "deepseek", "bailian", "kimi"]
  ```

  The provider is enabled by default when using GoModel's built-in defaults, but a
  custom config file overrides that list.
</Note>
